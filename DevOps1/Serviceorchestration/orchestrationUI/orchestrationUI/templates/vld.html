VLD DESCRIPTION


  • The VLD provides a description of each Virtual Link.
  • This type of information can be used by the NFVO to determine the appropriate placement of a VNF instance.
  • The VIM responsible for managing the virtualised resources of the selected placement to determine the allocation of required virtualised resources on a host with adequate network infrastructure.
  • The VIM, or another Network Controller, can also use this information to establish the appropriate paths and VLANs.
  • The VLD describes the basic topology of the connectivity (e.g. E-LAN, E-Line, E-Tree) between one or more VNFs connected to this VL and other required parameters (e.g. bandwidth and QoS class).
  • The VLD connection parameters are expected to have similar attributes to those used on the ports on VNFs in ETSI GS NFV-SWA 001 [i.8]. Therefore a set of VLs in a Network Service can be mapped to a Network Connectivity Topology (NCT) as defined in ETSI GS NFV-SWA 001
